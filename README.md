## Firebase production setup

### Required environment variables

Create a local `.env.local` based on `.env.example` and add the same values in Vercel project environment variables:

```bash
NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=

FIREBASE_PROJECT_ID=
FIREBASE_CLIENT_EMAIL=
FIREBASE_PRIVATE_KEY=
FIREBASE_STORAGE_BUCKET=
```

### Deploy security rules and indexes

```bash
npm run firebase:deploy:seguridad
```

You can also deploy each part:

```bash
npm run firebase:deploy:rules
npm run firebase:deploy:indexes
```

### Assign admin custom claim

```bash
npm run auth:set-admin -- <UID>
```

### Security hardening checklist

- Restrict Firebase Web API key by HTTP referrer (`localhost` + production domains).
- Restrict enabled Google APIs to only the Firebase APIs required by this app.
- Disable unused Firebase Auth providers (including anonymous if not needed).
